資料來源：bwin × 哈佛醫學院，2005–2007 年真實線上賭場行為數據
- Demographics（4,222 位用戶）：UserID、國籍、語言、性別、註冊日、首次入金日
- DailyAggregation（102,165 筆）：UserID、日期、當日投注額、當日彩金、當日下注次數

---

1. 識別長期讓賭場虧損的「職業套利客」（風控問題）

分析結果：
從 DailyAggregation 彙總每位用戶的 total_stake 與 total_winnings，計算 net_loss = total_stake - total_winnings。
net_loss 為負值代表玩家長期贏錢（賭場虧損），精準揪出前 1% 的持續獲利帳號。

潛在問題發現：
結合 Demographics，分析這些帳號是否集中在特定國籍、語言群體，
幫助風控團隊判斷是否存在特定地區的系統性套利行為，或賭場對特定玩家族群存在定價漏洞。

---

2. 分析連輸後的「凹單行為」——玩家是否在輸錢後加碼（商業營運問題）

分析結果：
以 DailyAggregation 的日序資料，計算每位用戶的每日 GGR（= Stake - Winnings）。
當 GGR > 0 代表玩家當日虧損，統計連續虧損 3 天、5 天後，下一日 Stake 的平均變化幅度。

潛在問題發現：
若玩家在連輸後平均 Stake 顯著上升，代表存在非理性凹單行為，
玩家錢包壽命面臨加速耗盡的風險，可建議在關鍵損失閾值介入（如發放體驗金、限制單日投注上限）以拉長玩家生命週期。

---

3. 玩家生命週期與流失風險分析（LTV 問題）

分析結果：
結合 Demographics 的 Fstpdate（首次入金日）與 DailyAggregation 的最後活躍日，
計算每位玩家的「活躍壽命（天數）」與「投注密度（活躍天數 / 總間隔天數）」。
依壽命長短進行分層，找出典型的短命玩家輪廓（國籍、性別、語言）。

潛在問題發現：
識別高流失風險的用戶族群，為營運團隊提供「何時、針對哪類玩家」進行留存干預的數據依據。

---

4. 資料工程對帳——Winnings 負值異常偵測（DE 問題）

分析結果：
Codebook 指出 Winnings 欄位存在負值（源自後端會計系統的對帳修正）。
透過 Python 清理階段自動標記所有 Winnings < 0 的記錄，統計異常筆數、涉及用戶數與時間分佈。

潛在問題發現：
若負值集中在特定時段或特定用戶，可能代表後端系統在高頻結算時存在延遲或重複入帳的 bug，
直接幫技術團隊定位「資料寫入管道的潛在漏洞」，展示 DE 思維而非純分析思維。

---

5. K-Means 玩家行為分群（ML 能力展示）

分析結果：
以每位用戶的行為特徵向量進行 K-Means 分群：
- total_stake（總投注規模）
- net_loss（損益表現）
- active_days（活躍天數）
- avg_daily_stake（平均每日投注額）
- bet_frequency（投注密度）

將玩家分為 3–4 群（如：高頻小額娛樂型 / 低頻大額鯨魚型 / 高風險凹單型 / 短命流失型）。

潛在問題發現：
不同分群對應不同營運策略（留存、限制、VIP 服務），
幫助產品團隊針對性設計玩家旅程，而非對全體用戶套用同一套規則。
